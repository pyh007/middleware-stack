# Elasticsearch 快照与状态恢复 Runbook

## 1. 先定义恢复目标

为数据集明确 RPO、RTO、允许的只读窗口、跨区域要求、加密和保留周期。区分误删索引、逻辑坏数据、节点丢失、整个集群丢失和版本升级失败；副本解决节点故障，不替代快照，快照也不自动补回快照后的写入。

## 2. 备份前检查

- 快照仓库必须由受控服务账号访问，凭据进入 keystore/secret manager，仓库网络与生产数据路径隔离。
- 监控快照开始/结束、失败分片、耗时、仓库容量和最近成功时间；避免多个集群同时写同一仓库。
- 记录 index template、component template、ILM、ingest pipeline、脚本、角色、别名和应用版本；是否包含 global state 必须是显式决策。
- 周期性运行仓库验证，并在隔离集群实际恢复；只检查快照 `SUCCESS` 不足以证明 RTO。

## 3. 隔离恢复步骤

1. 冻结相关写入并保存误操作时间线；若有可重放变更流，记录精确位点。
2. 创建兼容版本的隔离集群，确认目标版本支持源快照和索引创建版本。
3. 注册仓库为只读或确保只有一个写入者，列出快照与失败分片。
4. 先用 rename pattern 恢复到新索引名，避免覆盖仍可取证的数据。
5. 校验文档数、时间范围、业务主键抽样 checksum、mapping、settings、别名和权限。
6. 重放快照后的审计日志/CDC，处理幂等与版本冲突；再次记录水位和 checksum。
7. 让应用只读验证，再小流量读写；通过后原子切换别名或路由。

课程显式命令 `make -C Elasticsearch snapshot-restore` 只删除 `lab-es-07-restore`，演示快照、删除、恢复和 checksum；不能直接当作生产脚本。

## 4. 回滚与失败处理

恢复慢时先检查分片并行度、仓库吞吐、节点磁盘/网络和 recovery throttling，不直接无限提高并发。校验失败不切流，保留旧别名和原索引；清理失败恢复索引前保存日志。切换后若错误率、数据差异或写入冲突超过阈值，回切旧别名，并暂停双写或重放直到确定权威源。

## 5. 演练验收

记录快照点、声明 RPO、实际丢失窗口、开始/结束时间、实际 RTO、恢复字节/分片、校验样本、失败步骤和改进项。至少覆盖误删索引与整集群重建两类演练；版本升级前增加向新版本恢复的兼容性验证。
