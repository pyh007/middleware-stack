# Elasticsearch 安全、容量与升级 Runbook

## 安全基线

- 生产启用 TLS、身份认证和证书轮换，HTTP/transport 都限制网络暴露；不要使用本课程 HTTP 和默认密码。
- 应用账号按索引模式和动作授予最小权限，读写拆分；禁止长期使用 `elastic` 超级用户。
- 密码/API key 有 owner、用途、过期、轮换和吊销流程；审计创建、授权失败、敏感查询与管理操作。
- 快照仓库、Kibana、监控和自动化账号独立授权；日志脱敏，防止查询正文与凭据泄露。

[模块 07](../labs/07-snapshot-operations/README.md) 用临时只读用户证明“能读、不能写”，并在实验后删除账号和角色。

## 容量模型

用峰值写入字节/秒、保留期、副本数、索引膨胀系数和安全余量估算磁盘；额外预留 merge、recovery、watermark 与节点故障空间。用查询并发、聚合基数、分片 fan-out、缓存命中和 P99 估算 CPU/heap。目标分片尺寸不是固定答案，必须通过恢复速度、查询延迟和节点故障演练反推。

持续监控每节点/索引/分片的 docs、store、写入、搜索、segment、merge、heap/GC、磁盘、水位、thread pool 和 rejected。至少保留一个节点或故障域丢失时的容量余量；扩容计划要包含分片迁移给网络与磁盘造成的峰值。

## 变更和升级

升级前检查版本兼容、弃用、插件、客户端、快照仓库和索引创建版本；完成可恢复快照并在隔离环境验证。滚动升级一次只处理一个节点，等待节点加入、主副本恢复和 SLO 稳定后继续。升级期间冻结模板、mapping、ILM 和大规模迁移等无关变更。

灰度指标包括业务错误率/P99、节点离开时间、分片恢复速率、rejected、GC、磁盘与网络。无法在目标时间恢复、出现数据校验差异、错误率超阈值或新节点反复离群时停止推进。若版本不支持原地降级，应通过预先验证的快照恢复到旧集群并切回流量，而不是复用已被新版本写过的数据目录。

## 月度演练

每月复核账号/API key、证书到期、磁盘增长和分片偏斜；每季度恢复代表性快照并计时；重大版本升级前执行一个节点故障和一个仓库恢复。所有演练要写 RPO/RTO、证据、偏差和 owner。
