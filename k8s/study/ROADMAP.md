# Kubernetes 学习路线与进度

## 掌握等级

| 等级 | 完成标准 |
|---|---|
| L0 未学习 | 不能稳定解释对象和数据路径 |
| L1 能解释 | 能用自己的话说明机制与边界 |
| L2 能验证 | 能独立运行实验并指出可观察证据 |
| L3 能诊断 | 能根据状态、事件、日志和指标定位问题 |
| L4 能决策 | 能结合容量、风险和恢复目标说明生产取舍 |

## 八周主线

| 周次 | 主题 | 当前等级 | 最近复习 | 下次复习 | 必须保留的证据 |
|---|---|---:|---|---|---|
| 1 | 架构、API 与调谐 | L0 | - | - | generation、observedGeneration 和副本状态 |
| 2 | 工作负载与生命周期 | L0 | - | - | rollout 修订、Job 完成和探针行为 |
| 3 | 网络与服务发现 | L0 | - | - | Service、EndpointSlice、DNS 和真实响应 |
| 4 | 配置与安全 | L0 | - | - | 配置输出与 RBAC allow/deny |
| 5 | 存储与有状态工作负载 | L0 | - | - | PVC Bound、写入和跨工作负载读取 |
| 6 | 调度与资源治理 | L0 | - | - | requests/limits、QoS、调度节点和 Pending 事件 |
| 7 | 可观测性与排障 | L0 | - | - | 应用日志、Events 和诊断顺序 |
| 8 | 生产审计与 Kafka 事故 | L0 | - | - | 清单审计、事故时间线、恢复验证和回滚方案 |

## 每模块验收

- [ ] 三分钟内讲清对象职责、控制循环和失败边界。
- [ ] 不看答案独立运行安全实验并解释断言。
- [ ] 执行前能预测 `spec/status`、Endpoint 或调度结果。
- [ ] 使用显式命令制造一个故障，并按证据链修复。
- [ ] 回答关联面试题时包含结论、机制、证据、边界、副作用和生产验证。
- [ ] 写出监控、止损、验证和回滚步骤。
