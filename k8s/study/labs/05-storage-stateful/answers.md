# 存储与有状态参考答案

1. 先看 PVC Events 和 StorageClass 是否存在及默认选择，再看 provisioner、允许拓扑、容量和配额。若是 `WaitForFirstConsumer`，还要结合引用 PVC 的 Pod 调度约束判断，因为卷可能等到节点选择后才制备。
2. 不是。RWO 表示卷可以被单个节点以读写方式挂载；同一节点上的多个 Pod 是否可共享还取决于驱动和应用。跨节点写入通常需要 RWX 或应用级复制。
3. Pod 是消费者，删除后 PVC 通常仍在；删除 StatefulSet 通常也不会自动删除既有 PVC；删除 PVC 会释放申请并可能按回收策略删除底层卷；PV 的处理又受保护器、claimRef 和 reclaimPolicy 影响。必须查看实际对象，不能靠口诀执行删除。
4. StatefulSet 提供稳定序号、按序管理、稳定 DNS 配合和每副本 PVC 模板，但不自动复制数据、不处理选主正确性、不保证备份，也不会解决应用分区脑裂。
5. PVC 删除可能触发底层存储立即销毁，名称重建不能找回原数据。生产需使用准入保护、权限隔离、保留策略、快照和恢复演练降低风险。
6. 崩溃时的块级快照可能缺少应用一致性点，多个卷也可能时间不一致。需结合应用 flush、事务日志、备份校验，在隔离环境执行恢复并测量最大数据损失与恢复时间。
