# 主动回忆与间隔复习

复习先检索记忆，再查看答案。执行 `make -C k8s review` 随机抽题，口头按“结论 → 机制 → 证据 → 边界 → 副作用 → 生产验证”回答，然后打开题目关联的实验或 Runbook。`make -C k8s review-list` 可查看题库主题。

首次达到 L2 后安排第 1、3、7、14、30、90 天复习。每次只需：闭卷答三题、重跑一个最弱实验、修改一个变量、保存一条新证据、更新 [ROADMAP.md](../ROADMAP.md)。如果答案只剩术语，回到 README；如果原理能讲但不会诊断，运行显式故障实验；如果能诊断但不会决策，用综合事故写上线或恢复方案。

复习记录必须保留日期、题目、口述缺口、命令、证据、当前等级和下次日期。“看过文档”和“命令退出 0”都不是掌握证据；L3 至少要能从 status、Events、日志、EndpointSlice 或调度信息定位原因。
