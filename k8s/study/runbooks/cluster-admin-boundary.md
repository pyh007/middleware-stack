# 集群运维扩展线安全边界

节点 drain、容器运行时中断、控制平面升级、etcd 快照恢复和多节点网络故障会影响整个集群，不能在当前承载其他开发工作负载的 Docker Desktop 集群中作为默认课程运行。

执行这类训练前必须创建独立、可销毁的多节点集群，确认 kubeconfig 上下文和所有者，记录节点与数据范围，并准备从零重建方案。etcd 恢复需要验证快照版本、证书、成员关系和 API 对象一致性；节点升级需要遵守版本偏差规则，控制 PDB、容量余量和回滚窗口。

安全全量任务永远不依赖这些实验。将来新增显式命令时，应在命令名称、帮助和脚本开头列出影响范围、预计资源、数据损失可能性和清理步骤，并要求二次环境变量确认。完成后需验证业务、清理额外集群并恢复 kubeconfig 上下文。
