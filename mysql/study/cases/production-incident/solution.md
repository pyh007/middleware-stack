# 大促数据库故障参考思路

候选止损包括暂停非核心批处理、限制深分页入口、将写后读切回主库、对非核心接口限流或降级，并在评估事务业务影响后处理根阻塞。不要未保存现场就重启，也不要在副本延迟时贸然切换主库。

现场应同时保存业务延迟和错误、连接池、进程列表、当前与摘要 SQL、执行计划、锁等待图、长事务、InnoDB 状态、系统 CPU/IO/磁盘、复制线程和位点。根因很可能不是单点：不可索引的日期函数与深分页增加扫描，长事务扩大锁与连接占用，流量将排队进一步放大，副本延迟又破坏读一致性。

短期改写时间范围、限制页深、终止或拆分非核心长事务；长期使用稳定游标分页、查询驱动的联合索引、事务外移远程调用，并为连接池设置总预算和背压。索引上线要验证空间、IO、元数据锁、复制延迟、执行计划和回滚。

恢复能力不能依据任务状态判断。应在隔离实例恢复最新基线和日志链，校验核心订单、金额、约束和应用启动，记录真实 RPO/RTO。复盘措施必须有负责人、完成时间和再次演练的验收证据。
