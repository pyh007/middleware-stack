# MySQL 生产上线与巡检清单

## 每日或自动巡检

- 核心接口吞吐、错误率、P95/P99 延迟是否偏离基线。
- CPU、IO 延迟、磁盘空间和增长速度。
- 连接、运行线程、连接失败、锁等待、死锁和长事务。
- SQL 总耗时、扫描/返回比、计划突变和慢日志趋势。
- 副本线程、延迟、错误、binlog 保留和只读状态。
- 备份完成、文件完整性、异地副本和最近恢复演练结果。

## SQL 或索引上线前

- 使用代表性数据和真实参数验证结果一致。
- 保存优化前后 `EXPLAIN ANALYZE`、延迟和资源数据。
- 评估索引大小、写放大、临时空间、元数据锁和复制延迟。
- 明确灰度范围、成功指标、终止阈值和回滚耗时。

## 故障现场

- 先记录时间线、业务影响和最近变更。
- 同时采集连接、语句、锁、事务、InnoDB、系统和复制证据。
- 优先处理根阻塞和放大流量的入口，不只杀末端等待会话。
- 所有终止事务、切主、恢复和数据修复动作都要评估业务一致性。

## 定期演练

- 从独立备份恢复到隔离实例并启动应用验证。
- 模拟副本延迟、节点故障、误删和账号泄露。
- 记录实际 RPO/RTO，与业务目标比较并修正流程。
