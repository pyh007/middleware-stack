# Redis 学习路线与证据记录

## 掌握等级

| 等级 | 可验证标准 |
|---|---|
| L0 | 不能稳定解释术语之间的因果关系 |
| L1 | 能不用原文说明机制和适用边界 |
| L2 | 能独立运行实验、预测结果并解释证据 |
| L3 | 能根据 INFO、SLOWLOG、拓扑或恢复结果定位问题 |
| L4 | 能结合流量、数据价值、RPO/RTO 和成本做设计决策 |

## 间隔复习节奏

| 轮次 | 建议间隔 | 动作 | 升降级标准 |
|---|---:|---|---|
| R1 | 1 天 | 闭卷回答 3 题，复述关键数据路径 | 不能讲清边界则回到 L1 |
| R2 | 3 天 | 不看文档重跑一个实验 | 断言通过且能解释才保持 L2 |
| R3 | 7 天 | 从指标反推一个故障原因 | 能给诊断顺序才进入 L3 |
| R4 | 14 天 | 修改实验变量并预测副作用 | 预测与证据一致才保持等级 |
| R5 | 30 天 | 完成综合事故题和恢复方案 | 能定义验证与回滚才进入 L4 |
| R6 | 90 天 | 重新做容量与 RPO/RTO 决策 | 假设变化后仍能辩护才保持 L4 |

## 每模块验收

- [ ] 三分钟内讲清结论、机制、证据、边界和副作用。
- [ ] 不看答案独立跑通实验，并说明每个断言保护了什么。
- [ ] 修改指定变量后，先预测再解释实际差异。
- [ ] 给出监控、止损、长期修复、验证与回滚方法。
- [ ] 把实际日期和关键输出写入本文件，而不是只勾选阅读状态。
