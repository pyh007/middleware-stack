# 01 闭卷练习

运行实验前先写下答案和预测。

1. Hash 的逻辑类型没有变化，为什么字段增加后 `OBJECT ENCODING` 会变化？转换会影响哪些成本？
2. 用户对象存为一个 JSON String 与一个 Hash，各自在原子更新、内存、网络和演进上有什么取舍？
3. 排行榜为什么适合 ZSet？如果单榜有五千万成员，你会如何限制大 Key 和热点风险？
4. List 用作任务队列时缺少哪些可靠消费语义？什么时候应选择 Stream 或专业消息队列？
5. 预测实验中整数 String、短 String、长 String、小/大 Hash、小/大 ZSet 的编码。哪些结论依赖 Redis 版本和配置？
6. 修改 Hash 字段数或 ZSet 成员数，让转换边界前后各运行一次，记录编码和 `MEMORY USAGE`。
