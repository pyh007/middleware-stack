# 03 参考答案

1. Pipeline 是客户端批量传输，不承诺原子；MULTI/EXEC 将命令排队并连续执行；Lua 在服务端一次原子执行脚本。事务或 Lua 仍可能通过一次请求减少往返，但业务保证、失败处理和阻塞风险完全不同。
2. 单命令执行串行不等于全链路无等待。网络、DNS/连接、客户端池、超大响应、高复杂度命令、Lua、fork、磁盘、内存压力、主机调度、复制输出缓冲都可能抬高延迟。应按时间线取证而不是先猜根因。
3. 热 Key 看单位时间请求集中度，主要打 CPU/网卡/单槽；大 Key 看对象内存、元素数和响应大小，放大单命令、复制和迁移。热 Key 可复制读/本地缓存/分片，副作用是一致性；大 Key 可拆分，副作用是聚合和迁移复杂度。
4. SLOWLOG 只统计服务端命令执行时间，不含排队前的客户端池等待、网络、响应传输等。应检查应用分段耗时、连接池、超时重试、blocked clients、主机资源、网络和拓扑重定向。
5. 本机噪声、容器调度和连接复用会改变耗时；课程要证明语义和往返机制，不伪造稳定倍数。生产应多轮预热后看分位数和吞吐，并限制误差。
6. 固定总命令数、Value 大小、连接、主机和后台任务，只改变每批命令/字节数；观察客户端 P50/P99、吞吐、CPU、网络、内存和错误。达到业务延迟、缓冲或错误水位就停止，并保留旧批次作为回滚。
