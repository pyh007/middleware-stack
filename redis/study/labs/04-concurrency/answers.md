# 04 参考答案

1. `INCR` 的读、加、写在服务端一个命令内完成；分开的 GET 和 SET 之间可以插入其他客户端。实验用 Barrier 让 8 个线程都写入 1，稳定证明命令原子与业务流程原子的区别。
2. MULTI 阶段语法错误可能使 EXEC 放弃；执行阶段某条命令出错时其他命令仍可执行，Redis 不回滚已执行结果。需要业务补偿、幂等或 Lua 检查，不能套用数据库 ACID 口号。
3. WATCH 在热点键上会频繁失败并重试，增加 CPU、连接和尾延迟。要有最大重试、指数退避/抖动、业务排队或改成 Lua/单写者，并监控冲突率和降级结果。
4. Redis 事件循环执行 Lua 时不插入其他命令，因此检查库存与 DECR 原子。脚本不能调用外网，长循环会阻塞所有客户端；应限制 Key/数据量、预加载脚本并设置超时处置。
5. 持有者暂停到 TTL 过期后，新持有者获取同一键；旧持有者恢复后普通 DEL 会删除新锁。唯一 token 加 Lua 比较删除把检查和删除合成一个原子步骤，实验分别得到 0 和 1。
6. 即使正确解锁，旧进程仍可能在锁过期后继续操作外部资源。单调 fencing token 随每次获取递增，下游保存最大 token 并拒绝更小 token。生产需注入 GC/调度暂停和主从切换验证，而非只测正常路径。
