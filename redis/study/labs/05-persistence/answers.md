# 05 参考答案

1. RDB 的 RPO 取决于快照间隔，RTO 常较好；AOF everysec 将常见损失窗口缩到约秒级，但增加持续 I/O，恢复需重放；混合模式用 RDB 基础和 AOF 增量平衡恢复与窗口。实际保证还受磁盘、配置和故障类型影响，必须演练。
2. 子进程根据 fork 时视图生成新文件，主进程继续接收写并记录增量，完成后原子替换。风险包括 fork 暂停、页表、写时复制内存、重写缓冲和磁盘竞争。数据越大、写越密集风险越高。
3. 客户端成功通常表示内存执行完成；WAITAOF 可等待本地/副本 AOF fsync 条件；复制 offset 又是异步链路进度，三者边界不同。任何单一返回都不自动提供跨机房或业务端到端持久性。
4. 实验只证明一个受控 marker 在一次本机容器 SIGKILL、当前 everysec 配置和已有卷下恢复。它没有覆盖故障前最后一秒、宿主机/卷损坏、配置错误、文件损坏、全量恢复时长或跨系统对账。
5. 先隔离故障节点、保留卷/日志/配置，复制证据后用检查工具；在隔离实例加载最后可信文件，量化缺口，再从数据库或事件日志幂等补齐。核对总量、版本和关键记录后灰度切换，保留旧路由和快照以回滚。
6. 写入更多通常增加当前 AOF 和数据集，重写后 base 也增长；fork/COW/磁盘时间可能增加但不是线性固定倍数。停止条件应包括内存安全余量、磁盘延迟、业务 P99 和后台任务超时。
