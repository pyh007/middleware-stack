# 06 参考答案

1. 默认复制异步，成功只表示主内存执行；写可能尚未到副本或尚未持久化。故障转移选择的副本缺少该 offset 就会产生缺口。关键业务要明确可接受 RPO，必要时等待副本/持久化并从权威日志对账，但等待也不等于跨系统事务。
2. 单 Sentinel 判定不可达是主观下线；达到配置 quorum 才形成客观下线。发起故障转移的 Sentinel 还需获得多数授权，避免多个领导者。quorum=1 只为课程能在单 Sentinel 演示，不具备生产抗分区能力。
3. 旧主若保持可写，会与新主形成分叉；Sentinel 将其重配为副本以收敛拓扑。客户端若缓存旧地址，可能收到只读错误、连接失败，或在错误配置下产生双写，所以必须支持发现、退避和路由刷新。
4. CRC16 只计算非空花括号内容，所以相关键同槽，可执行多键命令/Lua并一起迁移。副作用是订单 42 的所有流量集中；Tag 粒度过粗会形成热点槽，需按业务聚合边界设计。
5. MOVED 表示槽的稳定所有者已改变，客户端应刷新槽缓存；ASK 是迁移过程的临时指向，下一条命令先发 ASKING，不应永久改映射。无限重试会放大故障，需有次数、退避和拓扑刷新。
6. 无副本时一个主节点故障会让其负责槽不可用，集群不能保持完整服务。生产至少为每主配置跨故障域副本，演练主故障、网络分区、客户端重定向、数据缺口和恢复时间，并验证不会把主从放在同一宿主机。
