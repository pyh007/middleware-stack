# 07 参考答案

1. 还需考虑 Key 对象、Value 对象、字典桶、类型编码/指针、过期表、allocator 对齐/碎片、复制积压、客户端缓冲和持久化 fork/COW。实验用 MEMORY USAGE 采样，但生产还要覆盖长尾和后台任务峰值。
2. 正常连接约 6000，发布峰值约 12000，再加运维、复制和安全余量；若 maxclients 不足，应先限制池、分批发布或扩拓扑，不能简单把 maxclients 调高而忽略文件描述符、内存和重连风暴。
3. used_memory 是 Redis 分配口径，RSS 是进程驻留页，maxmemory 是 Redis 数据/策略阈值，容器与主机是更外层硬限制。Redis 尚未到 maxmemory 也可能因 RSS、fork COW 或容器 OOM 被杀，需分层告警。
4. 为应用创建独立用户，只授权所需命令类别和例如 `~app:*` 的 Key Pattern；只读巡检、备份和管理员分离。轮换时先添加新凭据、灰度客户端、验证旧凭据无流量，再撤销旧凭据，并准备恢复窗口。
5. 它证明当前版本下一个对象能序列化、删除、重建并保持字段。没有证明全库文件健康、TTL/模块类型、Cluster 槽、跨对象一致性、恢复时长或业务缺口可补。全量恢复必须隔离演练和对账。
6. 缓存通常可从源站重建，RPO 可较宽但 RTO 受源站容量限制；会话可能允许分钟级损失但需业务确认；幂等令牌丢失会导致重复副作用，应以数据库唯一约束/业务流水为权威。数值必须结合业务损失，不存在通用答案。
