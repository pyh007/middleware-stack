# 08 参考答案

1. 穿透集中在不存在或非法 Key，每次未命中；击穿集中在少数热点到期瞬间；雪崩是大量 Key/节点同窗失效。证据分别看不存在查询回源、单键并发重建和 TTL/整体命中分布，不能仅凭“数据库 QPS 高”命名。
2. 空值缓存简单，但新数据创建后在短 TTL 内可能仍被视为不存在，还占额外键；布隆过滤器节省查询但有假阳性、需构建/更新/重建且不能单独证明数据存在。都需旁路校验和过期策略。
3. 获取不到锁的请求可短暂等待后重读、返回允许的陈旧值或降级，不能全部无界自旋。锁需 TTL、唯一 token、原子释放和重建指标；持有者失败后由下一请求接管，并限制源站并发。
4. 抖动上限不能突破业务有效期，下限要足以打散峰值，可按历史刷新量和源站安全 QPS计算。节点宕机会让其全部键同时不可用，与每键 TTL 无关，仍需高可用、限流和降级。
5. 更新 2 先回填，更新 1 的慢请求后回填会倒退。Lua 读取当前版本，仅在 incoming 更大时写入；版本必须来自数据库单调序列而非不可靠客户端时钟。实验中 version 1 返回 0，version 3 返回 1。
6. 清库会把所有命中变为回源，全量预热又制造集中读写，在数据库已过载时形成正反馈。先限流/降级、关闭放大重试、读允许的陈旧值、按热点小批限速预热，再修 TTL/大 Key/路由；每步观察数据库水位和错误率并可回滚。
