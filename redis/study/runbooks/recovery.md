# Redis 备份与状态恢复 Runbook

## 目标与前提

先按数据价值定义 RPO/RTO。例如缓存可用源站重建，RPO 可放宽；幂等令牌、会话或业务状态不能把 Redis 当作无损事实源。记录 RDB 时间、AOF fsync 策略、复制拓扑、故障时刻和客户端最后确认写入，避免把“文件存在”误当成可恢复。

## 标准步骤

1. 隔离故障节点并保留原卷、日志和配置，只在副本上制作证据副本。
2. 校验 RDB/AOF 工具检查结果和文件时间，明确理论数据缺口。
3. 在隔离实例加载备份，禁止直接覆盖线上当前卷。
4. 用关键键、记录数、版本/时间范围和业务总量验证；敏感数据只输出汇总。
5. 从数据库、事件日志或审计流水补齐故障窗口，重放必须幂等且限速。
6. 小流量切换并观察错误、延迟、命中率、复制和持久化，再逐步扩大。

## 回滚与验收

切换前保留旧路由和只读快照；若校验失败，停止新增写入恢复实例，路由回已知健康源并继续降级。验收至少包含实际丢失窗口、恢复耗时、关键记录抽样/总量对账、客户端重连和一次重启复验。课程中的 [DUMP/RESTORE 实验](../labs/07-production-ops/README.md) 验证对象级恢复，[SIGKILL 实验](../labs/05-persistence/README.md) 验证 AOF 重启恢复；它们不能替代生产全量恢复演练。
