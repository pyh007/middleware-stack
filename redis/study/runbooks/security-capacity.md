# Redis 安全与容量上线清单

## 容量

- 用真实键样本的 `MEMORY USAGE`、元素数和 TTL 分布估算，不用序列化字节数代替内存。
- 加上增长周期、碎片、复制积压、客户端输出缓冲、AOF rewrite/RDB fork 峰值和至少 30% 安全余量。
- 记录 `maxmemory` 与淘汰策略；确认哪些键有 TTL、哪些键允许被淘汰，并压测源站承受淘汰后的回源。
- 连接预算为应用实例数 × 每实例连接池上限，加运维和复制连接，必须低于 `maxclients` 且考虑发布双倍实例。

## 安全

- 只暴露必要网络，使用 TLS/私网/防火墙；禁止把本课程密码用于任何共享环境。
- 用 ACL 为应用、只读巡检、备份和管理员拆分用户与命令/Key Pattern，遵循最小权限。
- 密码通过 Secret 管理并演练双凭据轮换；日志和诊断输出不得泄露 AUTH、DUMP 内容或用户数据。
- 记录配置变更、危险命令与管理员访问；限制或重命名危险命令不能替代网络和身份控制。

## 上线、验证与回滚

上线前跑容量峰值、淘汰、持久化、主从延迟和故障转移测试。灰度时监控业务错误/P99、命中率、内存、碎片、淘汰、连接、复制 lag 和磁盘。为 maxmemory、淘汰策略、ACL、连接池和路由分别准备旧值与回滚负责人；任何正确性异常优先停止写入并从权威源对账。
